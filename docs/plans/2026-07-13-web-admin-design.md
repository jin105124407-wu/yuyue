# 电脑端网页后台第一版设计

## 目标

为 Moyo Beauty 预约小程序增加一个电脑端网页后台，让店内电脑可以登录后台、查看和处理预约、维护项目/美容师，并通过电脑音响播放新预约/取消预约提示音。

## 方案取舍

采用“静态网页 + 专用云函数 API”的第一版方案。

- 静态网页放在 `web-admin/`，可以本地打开，也可以后续部署到 CloudBase 静态托管。
- 专用云函数放在 `cloudfunctions/webAdminApi/`，不复用小程序 `OPENID` 鉴权，因为电脑网页没有小程序 openid。
- 登录方式使用管理员手机号 + 后台密码。管理员手机号仍来自 `admins` 集合。

## 鉴权设计

`admins` 集合支持以下字段：

```json
{
  "phone": "13600000000",
  "name": "店主",
  "webPassword": "临时明文密码"
}
```

也支持推荐的哈希字段：

```json
{
  "phone": "13600000000",
  "name": "店主",
  "webPasswordHash": "sha256后的密码"
}
```

第一版为了便于上线，兼容 `webPassword` 明文密码。后续可移除明文支持，只保留哈希。

登录成功后，云函数返回一个 12 小时有效的签名 token。网页把 token 存在浏览器本地存储中，后续请求带 token。

## 功能范围

第一版包含：

- 今日统计：今日预约数、顾客数、完成收入。
- 订单管理：按状态查看订单，确认预约、标记完成、取消预约、标记已读。
- 项目管理：新增/编辑/停用服务项目。
- 美容师管理：新增/编辑/停用美容师。
- 预约播报：浏览器轮询待播报预约，播放提示音，并把播报状态标记为已播放。

暂不包含：

- 微信扫码网页登录。
- 多角色权限。
- 复杂报表。
- 图片上传管理。
- 公网域名和 HTTPS 证书绑定自动化。

## 数据流

浏览器后台向 `webAdminApi` 发送 JSON 请求：

```json
{
  "action": "listOrders",
  "token": "...",
  "status": "pending"
}
```

`webAdminApi` 校验 token 后访问云数据库，并返回统一结构：

```json
{
  "ok": true,
  "data": {}
}
```

登录请求不需要 token：

```json
{
  "action": "login",
  "phone": "13600000000",
  "password": "后台密码"
}
```

## 界面设计

电脑端后台采用工作台布局：

- 左侧导航：总览、订单、项目、美容师、播报。
- 顶部：当前门店、登录管理员、刷新/退出。
- 主区域：紧凑表格和表单，适合电脑反复操作。
- 播报区：保持浏览器打开即可轮询，检测新预约后播放音频。

视觉上走安静、清晰、护理门店风格，不做营销落地页。
